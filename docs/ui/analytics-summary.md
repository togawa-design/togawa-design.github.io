## 1. GA4（Google Analytics 4）で収集しているデータ

### 基本指標

| 指標 | 説明 | 用途 |
|------|------|------|
| ページビュー (PV) | ページが表示された回数 | アクセス量把握 |
| ユニークユーザー (UV) | 訪問者数（重複排除） | 実際の訪問者数把握 |
| セッション数 | 訪問セッションの数 | 訪問回数把握 |
| 平均滞在時間 | ページでの滞在時間 | コンテンツ品質評価 |
| 直帰率 | 1ページのみで離脱した割合 | ページ品質評価 |

### 人口統計

| 指標 | 説明 | 用途 |
|------|------|------|
| 性別 | 男性/女性の割合 | ターゲット分析 |
| 年齢層 | 18-24/25-34/35-44/45-54/55-64/65+ | ターゲット分析 |
| 地域 | 都道府県別アクセス | エリアマーケティング |

### 流入元

| 流入元 | 説明 |
|--------|------|
| Organic Search | Google/Yahoo等の自然検索 |
| Direct | URL直接入力・ブックマーク |
| Referral | 他サイトからのリンク |
| Social | SNSからの流入 |
| Paid Search | 検索広告からの流入 |

### 表示画面
- 概要ダッシュボード
- アナリティクス詳細 > トラフィックタブ
- アナリティクス詳細 > トレンドタブ

## 2. 独自ページ計測（page-analytics.js）

LP（lp.html）と採用ページ（company-recruit.html）で独自にデータを収集しています。

### 収集イベント

#### 2.1 ページビュー (page_view)

| フィールド | 説明 | 例 |
|-----------|------|-----|
| pageType | ページ種別 | `lp` / `recruit` |
| companyDomain | 会社識別子 | `example-corp` |
| jobId | 求人ID（LPのみ） | `job001` |
| visitorId | 訪問者識別子 | `abc123...` |
| sessionId | セッション識別子 | `xyz789...` |
| device | デバイス種別 | `mobile` / `tablet` / `desktop` |
| screenWidth | 画面幅 | `375` |
| screenHeight | 画面高さ | `812` |
| userAgent | ブラウザ情報 | `Mozilla/5.0...` |
| referrer | リファラURL | `https://google.com/...` |
| referrerType | 流入元分類 | `google` / `indeed` / `line` 等 |
| utm_source | UTMソース | `google` |
| utm_medium | UTMメディア | `cpc` |
| utm_campaign | UTMキャンペーン | `spring_2026` |
| utm_content | UTMコンテンツ | `banner_a` |
| utm_term | UTMキーワード | `東京 求人` |

#### 2.2 CTAクリック (cta_click)

| フィールド | 説明 | 例 |
|-----------|------|-----|
| buttonType | ボタン種別 | `apply` / `phone` / `line` / `form` |
| buttonLocation | ボタン位置 | `hero` / `footer` / `sticky` |
| その他 | ページビューと同じフィールド | - |

#### 2.3 スクロール深度 (scroll_depth)

| フィールド | 説明 | 例 |
|-----------|------|-----|
| depth | スクロール到達率 | `25` / `50` / `75` / `100` |
| その他 | ページビューと同じフィールド | - |

#### 2.4 滞在時間 (time_on_page)

| フィールド | 説明 | 例 |
|-----------|------|-----|
| durationSeconds | 滞在秒数 | `45` |
| その他 | ページビューと同じフィールド | - |

#### 2.5 応募完了 (application_complete)

| フィールド | 説明 | 例 |
|-----------|------|-----|
| applicationType | 応募種別 | `web` / `phone` / `line` |
| その他 | ページビューと同じフィールド | - |

### 流入元分類（referrerType）

| 分類 | 判定条件 |
|------|----------|
| `direct` | リファラなし |
| `google` | google.com/google.co.jp |
| `yahoo` | yahoo.co.jp/search.yahoo |
| `indeed` | indeed.com/jp.indeed.com |
| `jobbox` | jobbox.me |
| `twitter` | twitter.com/t.co/x.com |
| `facebook` | facebook.com/fb.com |
| `line` | line.me/liff.line.me |
| `internal` | 自サイト内遷移 |
| `other` | その他 |

## 3. Firestoreで管理しているデータ

### 3.1 応募データ（applications コレクション）

| フィールド | 説明 | 分析用途 |
|-----------|------|----------|
| status | ステータス | 採用ファネル分析 |
| statusHistory | ステータス変更履歴 | リードタイム計算 |
| companyDomain | 会社 | 企業別集計 |
| jobId | 求人 | 求人別集計 |
| assignedTo | 担当者 | 担当者別パフォーマンス |
| createdAt | 応募日時 | 期間集計 |
| updatedAt | 更新日時 | 対応速度計算 |

### 3.2 ステータス種別

```
new (新規応募)
    │
    ▼
contacted (連絡済み)
    │
    ▼
interviewing (面接調整中)
    │
    ▼
offered (内定)
    │
    ▼
hired (入社)

※ rejected (不採用) / withdrawn (辞退) に分岐可能
```

### 3.3 statusHistory の構造

```json
{
  "statusHistory": [
    {
      "status": "new",
      "timestamp": "2026-02-10T10:00:00Z",
      "previousStatus": null
    },
    {
      "status": "contacted",
      "timestamp": "2026-02-10T14:30:00Z",
      "previousStatus": "new"
    },
    {
      "status": "interviewing",
      "timestamp": "2026-02-12T09:00:00Z",
      "previousStatus": "contacted"
    }
  ]
}
```

## 4. 分析機能の詳細

### 4.1 採用管理タブ【Admin専用】

#### 対応アラート
- **目的**: 未対応応募者の早期検出
- **条件**: `status = new` かつ `createdAt` から48時間以上経過
- **表示**: 応募者名、会社名、経過時間

#### 採用ファネル
- **目的**: 選考プロセスの可視化
- **指標**: 各ステータスの人数と通過率
- **計算**: 累積カウント方式（contacted以上→連絡済に含む）

```
応募 (100) → 連絡済 (80) → 面接 (50) → 内定 (20) → 入社 (15)
              80%          62.5%       40%         75%
```

#### リードタイム統計

| 指標 | 説明 | 計算方法 |
|------|------|----------|
| 初回レスポンス | 応募から連絡までの時間 | new → contacted の時間差 |
| 面談設定 | 連絡から面談調整までの時間 | contacted → interviewing |
| 選考判断 | 面談から内定までの時間 | interviewing → offered |
| 採用全体 | 応募から入社までの時間 | new → hired |

#### リードタイム分布グラフ

| バケット | 範囲 |
|----------|------|
| 0-6h | 6時間以内 |
| 6-12h | 6〜12時間 |
| 12-24h | 12〜24時間 |
| 24-48h | 24〜48時間 |
| 48h+ | 48時間以上 |

#### 担当者別パフォーマンス

| 指標 | 説明 |
|------|------|
| 担当数 | 割り当てられた応募者数 |
| 平均レスポンス | 初回連絡までの平均時間 |
| 採用率 | 担当応募者のうち入社した割合 |
| 承諾率 | 内定者のうち入社した割合 |

#### 企業別ドリルダウン
- 企業をクリックで詳細表示
- 企業別ファネル、リードタイム、担当者パフォーマンス

### 4.2 企業別タブ【Admin専用】

| 指標 | 説明 |
|------|------|
| PVランキング | ページビュー数順の企業ランク |
| 応募数ランキング | 応募数順の企業ランク |
| CVR | 企業別コンバージョン率 |

### 4.3 LP/採用ページタブ

#### ページ種別比較

| 指標 | LP | 採用ページ |
|------|:--:|:---------:|
| PV | ○ | ○ |
| UV | ○ | ○ |
| CTAクリック数 | ○ | ○ |
| 平均滞在時間 | ○ | ○ |
| CVR | ○ | ○ |

#### 流入元分析

10種類の流入元別に以下を集計:
- PV数
- UV数
- CTAクリック数
- CVR

#### デバイス分析

| デバイス | 集計項目 |
|----------|----------|
| モバイル | PV/UV/割合 |
| タブレット | PV/UV/割合 |
| デスクトップ | PV/UV/割合 |

### 4.4 コンバージョンタブ

| 指標 | 説明 |
|------|------|
| 全体CVR | 応募数 / PV |
| LP CVR | LP経由の応募率 |
| 採用ページCVR | 採用ページ経由の応募率 |
| ファネル | PV → CTA → 応募 の推移 |

### 4.5 トラフィックタブ

| 指標 | 説明 |
|------|------|
| 流入元詳細 | 各流入元のPV/UV/セッション |
| ランディングページ | 最初に訪問したページの分布 |
| 参照元URL | リファラURLの詳細 |

### 4.6 トレンドタブ

| 期間 | 表示内容 |
|------|----------|
| 日別 | 過去30日の日次推移 |
| 週別 | 過去12週の週次推移 |
| 月別 | 過去12ヶ月の月次推移 |

表示指標:
- PV推移
- UV推移
- 応募数推移
- CVR推移
