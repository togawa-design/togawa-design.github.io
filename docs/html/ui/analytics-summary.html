4. 分析機能の詳細

4.1 採用管理タブ【Admin専用】

#### 対応アラート

  • 目的: 未対応応募者の早期検出
  • 条件: status = new かつ createdAt から48時間以上経過
  • 表示: 応募者名、会社名、経過時間
#### 採用ファネル
  • 目的: 選考プロセスの可視化
  • 指標: 各ステータスの人数と通過率
  • 計算: 累積カウント方式（contacted以上→連絡済に含む）
応募 (100) → 連絡済 (80) → 面接 (50) → 内定 (20) → 入社 (15)
              80%          62.5%       40%         75%

#### リードタイム統計

指標説明計算方法
初回レスポンス応募から連絡までの時間new → contacted の時間差
面談設定連絡から面談調整までの時間contacted → interviewing
選考判断面談から内定までの時間interviewing → offered
採用全体応募から入社までの時間new → hired
#### リードタイム分布グラフ

バケット範囲
0-6h6時間以内
6-12h6〜12時間
12-24h12〜24時間
24-48h24〜48時間
48h+48時間以上
#### 担当者別パフォーマンス

指標説明
担当数割り当てられた応募者数
平均レスポンス初回連絡までの平均時間
採用率担当応募者のうち入社した割合
承諾率内定者のうち入社した割合
#### 企業別ドリルダウン
  • 企業をクリックで詳細表示
  • 企業別ファネル、リードタイム、担当者パフォーマンス
